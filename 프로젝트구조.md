# Poker Q-Learning 프로젝트 구조

대학원 강화학습 과목 기말 프로젝트. 헤즈업(1대1) 노리밋 텍사스 홀덤에서 테이블 기반 Q-러닝을 두 가지 알고리즘으로 학습·비교한다.

## 폴더 구조

```
minimizing/
├── .venv/                          Python 3.13 가상환경 (pokerkit, treys 설치됨)
│
├── poker-pokerkit/                 TD(0) bootstrap 버전
│   ├── abstraction.py              PokerKit state → (Round, Position, State, Action) 변환
│   ├── qlearning.py                QLearning 클래스 (update_q: TD 벨만 업데이트)
│   ├── train.py                    단일 학습 스크립트 (vs RandomAgent, 40,000 에피소드)
│   └── train_eval.py               학습 + 주기적 평가 (vs Random / vs RuleBased, CSV 저장)
│
├── poker-pokerkit-mc/              Monte Carlo + 즉각 비용 버전
│   ├── abstraction.py              (TD 버전과 동일)
│   ├── qlearning.py                QLearning 클래스 (update_mc: MC 역전파 업데이트)
│   ├── train.py                    단일 학습 스크립트
│   ├── train_eval.py               학습 + 주기적 평가 (CSV 저장: eval_results_mc.csv)
│   └── TD와MC비교.md               두 알고리즘 설계·실험·해석 보고서 (8개 섹션)
│
├── 평가지표_mbb.md                 mbb/g 평가 지표 출처·계산·해석 정리
└── 프로젝트구조.md                  (본 파일)
```

## 핵심 컴포넌트

### 추상화 계층 (`abstraction.py`)

| Enum | 값 | 비고 |
|---|---|---|
| `Round` | PREFLOP, FLOP, TURN, RIVER | PokerKit `street_index` 기반 |
| `Position` | BB(0), SB(1) | PokerKit 헤즈업: player 0 = BB, player 1 = SB |
| `State` | PREMIUM, STRONG, GOOD, DECENT, MEDIOCRE, WEAK, POOR, TRASH | 핸드 강도 8버킷 (프리플랍: Chen formula, 포스트플랍: treys Evaluator) |
| `Action` | FOLD, CHECK, CALL, RAISE_25/50/75/100, RAISE_ALLIN | 팟 % 기반 베팅 5종 + 폴드/체크/콜 |

상대 변환 함수: `pk_to_round`, `pk_to_position`, `pk_to_state`, `legal_our_actions`, `execute_action`.

### Q-테이블 (`qlearning.py`)

- **크기**: Round(4) × Position(2) × State(8) × Action(8) = **512 엔트리**
- **시그니처**: `Q(r, p, s, a) → float`
- **TD 버전**: `update_q(r, p, s, a, reward, next_r, next_s, terminal)` — Bellman bootstrap
- **MC 버전**: `update_mc(r, p, s, a, G)` — 에피소드 종료 후 역방향 누적 리턴 G로 업데이트

### 평가 상대 (`train_eval.py`)

두 종류 상대를 200게임씩 대전:
- **RandomAgent**: 합법 행동 카테고리 중 균등 선택
- **RuleBasedAgent**: (라운드 × facing-bet × 핸드 강도 8단계) 기반 고정 정책. 강한 패는 라운드별 차등 비율로 베팅, 약한 패는 직면 시 폴드. 자세한 표는 [poker-pokerkit-mc/TD와MC비교.md](poker-pokerkit-mc/TD와MC비교.md) 1절 참조.

학습·평가 모두 매 에피소드마다 학습 에이전트의 포지션을 BB/SB로 교대해 한쪽 포지션 편향을 방지한다. 동시에 포지션이 Q-테이블 차원에 포함되어 있어 두 포지션의 경험이 별도 셀에 저장된다.

## 실행 방법

```powershell
# 가상환경 활성화
& c:\code\minimizing\.venv\Scripts\Activate.ps1

# TD(0) 버전
cd c:\code\minimizing\poker-pokerkit
python train_eval.py           # 학습 + 평가 → eval_results.csv

# MC + 즉각 비용 버전
cd c:\code\minimizing\poker-pokerkit-mc
python train_eval.py           # 학습 + 평가 → eval_results_mc.csv
```

각 스크립트는 40,000 에피소드 학습 동안 200마다 평가를 수행하고, 결과를 CSV로 저장한 뒤 최종 Q-테이블을 콘솔에 출력한다.

## 하이퍼파라미터

| 항목 | 값 |
|---|---|
| 총 학습 에피소드 | 40,000 |
| 학습률 α | 0.1 |
| 할인율 γ | 0.9 |
| 탐색 ε | 1.0 → 0.05 (선형, 80% 진행 시점 도달) |
| 평가 주기 | 200 에피소드 |
| 평가 게임 수 | 각 상대 200 게임 |
| 시작 스택 | 200 칩 |
| 블라인드 | 1 / 2 |

## 두 알고리즘 비교

| 항목 | TD(0) | MC + 즉각 비용 |
|---|---|---|
| 업데이트 시점 | 매 액션 직후 | 에피소드 종료 후 일괄 |
| 베팅 비용 반영 | 즉시 반영 없음 (방문 반복으로만 간접 전파) | 액션 즉시 반영 |
| 승리 신호 전파 | 반복 방문이 필요 | 에피소드 1회에 전체 전파 |
| Bootstrap 사용 | 사용 | 미사용 (실제 결과만) |
| 분산 / 편향 | 낮음 / 높음 | 높음 / 낮음 |

자세한 설계·실험·해석은 [poker-pokerkit-mc/TD와MC비교.md](poker-pokerkit-mc/TD와MC비교.md) 참조. 평가 지표(mbb/g, 표준오차, 신뢰구간) 의 출처와 계산 방법은 [평가지표_mbb.md](평가지표_mbb.md) 참조.
