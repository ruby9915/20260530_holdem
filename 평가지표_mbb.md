# 평가 지표: mbb/g (milli big blinds per game)

본 프로젝트에서 학습 에이전트의 성능을 측정하는 주 지표.

## 1. 출처

**mbb/g** 는 컴퓨터 포커 연구의 표준 평가 단위로, 학계와 산업계 양쪽에서 널리 사용된다.

- **학계**: ACPC (Annual Computer Poker Competition, 2006–2018) 가 공식 평가 단위로 채택. AAAI·IJCAI 등의 포커 AI 논문 (DeepStack, Libratus, Pluribus 등) 이 모두 mbb/g 로 결과를 보고함.
- **산업계**: 온라인 포커 분석 도구 (PokerTracker, Hold'em Manager) 는 동일한 개념을 **bb/100** (100핸드당 BB 수익) 단위로 표시. 1 bb/100 = 10 mbb/g 로 본질이 같다.

### 왜 이 단위를 쓰는가

포커 성능 비교에서 다음 두 문제를 동시에 해결해야 한다.

1. **블라인드·스택 크기 무관 비교**: 게임마다 블라인드가 1/2 일 수도, 5/10 일 수도 있다. 칩 단위 평균은 직접 비교가 불가능하다. BB 로 정규화하면 환경에 무관하게 같은 척도가 된다.
2. **소수점 표현 회피**: BB 단위 그대로 쓰면 "0.0123 BB/game" 같은 작은 수가 자주 나온다. 1000을 곱해 mbb 로 표시하면 "12.3 mbb/g" 처럼 정수에 가까운 읽기 좋은 숫자가 된다.

### 직관적 해석 기준 (포커 커뮤니티 통념)

| mbb/g | bb/100 | 의미 |
|---|---|---|
| +50 이상 | +5 이상 | 매우 강한 winning player |
| +10 ~ +50 | +1 ~ +5 | winning player |
| -10 ~ +10 | -1 ~ +1 | break-even (본전) |
| -50 이하 | -5 이하 | losing player |

본 프로젝트의 평균 payoff 단위가 칩(예: -1)이라도 1 칩 = 500 mbb (BB=2) 이므로, mbb 로 환산하면 ±수백 ~ 수천 범위가 나온다.

## 2. 계산 방법

### 2.1 한 게임의 결과 (payoff)

평가 한 게임마다 학습 에이전트의 칩 수익 하나가 기록된다.

$$
\text{payoff}_i = (\text{게임 종료 후 스택}) - (\text{초기 스택 200})
$$

본 프로젝트에서는 200게임 또는 500게임 평가를 수행하므로, 평가 1회당 payoff 가 N개 수집된다.

### 2.2 mbb 변환

각 게임의 payoff 를 BB 단위로 정규화한 뒤 1000을 곱한다.

$$
\text{mbb}_i = \frac{\text{payoff}_i}{\text{BB}} \times 1000
$$

본 프로젝트는 BB=2 이므로:

$$
\text{mbb}_i = \text{payoff}_i \times 500
$$

### 2.3 평균 (mbb/g)

게임당 평균 수익:

$$
\overline{\text{mbb/g}} = \frac{1}{N} \sum_{i=1}^{N} \text{mbb}_i
$$

이것이 콘솔 출력의 `mbb/g` 자리에 표시되는 값.

### 2.4 표준편차 (s)

게임별 결과가 평균에서 흩어진 정도. 표본 표준편차 (자유도 N-1):

$$
s = \sqrt{\frac{1}{N-1} \sum_{i=1}^{N} (\text{mbb}_i - \overline{\text{mbb/g}})^2}
$$

포커는 게임당 결과의 분산이 매우 크다 (이긴 판은 +1000 mbb 수준, 진 판은 -10000 mbb 수준이 흔함). 따라서 s 가 평균보다 훨씬 큰 것이 정상이다.

### 2.5 표준오차 (SE) — 평균의 불확실성

표준편차 s 는 **개별 게임**의 흩어짐이지만, 실제로 알고 싶은 것은 "이 평균값 자체가 얼마나 신뢰할 만한가" 이다. 중심극한정리에 따라 표본 평균의 표준편차는 다음과 같이 계산된다.

$$
\text{SE} = \frac{s}{\sqrt{N}}
$$

이것이 콘솔 출력의 `±` 뒤에 표시되는 값.

**핵심 직관**: 평가 게임 수 N 을 4배로 늘리면 SE 는 절반으로 줄어든다 (√4 = 2). 즉 표본을 늘릴수록 평균 추정이 더 정확해진다.

### 2.6 신뢰구간 (CI)

표본 평균이 정규분포를 따른다는 가정 (N 이 충분히 크면 성립) 하에:

$$
\text{95\% CI} = \overline{\text{mbb/g}} \pm 1.96 \times \text{SE} \approx \overline{\text{mbb/g}} \pm 2 \cdot \text{SE}
$$

$$
\text{68\% CI} = \overline{\text{mbb/g}} \pm 1 \cdot \text{SE}
$$

**해석**: 동일한 실험을 반복하면 95% 의 경우 진짜 평균이 이 구간 안에 들어온다.

## 3. 실전 해석 예시

### 예시 1: 명백한 흑자

```
mbb/g = +19150 ± 2848
```

- 95% CI = [19150 - 5696, 19150 + 5696] = **[+13454, +24846]**
- 구간이 0 에서 한참 떨어져 있음 → 통계적으로 명백한 흑자

### 예시 2: 노이즈 (결론 불가)

```
mbb/g = -694 ± 401
```

- 95% CI = [-694 - 802, -694 + 802] = **[-1496, +108]**
- 0 을 포함 → "진짜 평균이 본전일 가능성" 을 배제할 수 없음
- 더 많은 게임으로 검증 필요

### 예시 3: 승률은 높지만 적자

```
승률 96% · mbb/g = -183 ± 866
```

- 승률만 보면 압도적이지만, 95% CI = [-1915, +1549] 로 0 을 크게 포함
- "작은 팟 96개 이기고 큰 팟 4개를 크게 잃는" 패턴이 mbb/g 에 잡힘
- 승률과 mbb/g 가 서로 다른 정보를 준다는 증거

## 4. 코드 구현

[poker-pokerkit-mc/train_eval.py](poker-pokerkit-mc/train_eval.py) (TD 버전도 동일):

```python
def _mbb_and_se(payoffs: list[float]) -> tuple[float, float]:
    n = len(payoffs)
    mean = sum(payoffs) / n              # 평균 (칩 단위)
    std  = statistics.stdev(payoffs)     # 표본 표준편차 (N-1)
    scale = 1000.0 / BIG_BLIND           # 칩 → mbb 변환 계수
    return mean * scale, (std / math.sqrt(n)) * scale
    #                    └─────────────┘
    #                    SE = s / √N
```

스케일 계수 `1000/BB` 는 평균과 SE 양쪽에 동일하게 곱해진다 (선형 변환).

콘솔 출력 형식:

```
 episode │ win%_rand     mbb/g_rand │ win%_rule     mbb/g_rule
       0 │     18.2%     -475±  23 │     40.6%     -114±  26
     500 │     85.8%   +15187±2455 │     90.0%    -2588± 779
```

CSV 컬럼: `win%_vs_random`, `mbb/g_vs_random`, `se_vs_random`, `win%_vs_rulebased`, `mbb/g_vs_rulebased`, `se_vs_rulebased`.

## 5. 승률 (win%) 과의 관계

승률은 보조 지표로 함께 출력하지만, mbb/g 와 다음과 같이 다르다.

| 항목 | 승률 | mbb/g |
|---|---|---|
| 측정 대상 | 이긴 판의 비율 | 게임당 평균 수익 |
| 큰 손실 반영 | 1패로만 카운트 | 절대 금액 그대로 반영 |
| 큰 승리 반영 | 1승으로만 카운트 | 절대 금액 그대로 반영 |
| 진짜 실력 지표 | 부분적 | 직접적 |

포커에서는 "승률 99% 인데 적자" 가 가능하다 (작은 판 99개 이기고 한 판 큰 올인으로 잃으면). 따라서 **mbb/g 가 최종 판단 기준**이고, 승률은 정책의 공격성·보수성을 정성 분석할 때 참고용으로 사용한다.
