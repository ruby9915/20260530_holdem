27번 실험 — 강화학습 환경 설계

혼합/순환 페르소나 학습을 통한 분포 강건성 (Heads-Up No-Limit Texas Hold'em) · PPT 발표용 정리 자료

한 줄 요약 — "2,048셀 이산 상태 × 8행동 / 몬테카를로 비례배분 보상 / Softmax 탐색" 위에서, 학습 상대를 5종 페르소나 혼합으로 다양화해 단일 상대 과적합을 깨고 분포 강건성을 입증한 실험.

0. 게임 환경

  • 엔진: pokerkit No-Limit Texas Hold'em, 헤즈업(2인) 구조
  • 스택 / 블라인드: 시작 스택 200, 빅블라인드 2, 스몰블라인드 1 → 한 핸드당 100 BB 깊이
  • 매 에피소드 = 1 핸드 (딜 → 베팅라운드 → 쇼다운/폴드까지)

1. 상태 (s) — 4차원 추상화

연속적인 포커 상황을 2,048개 셀로 이산화한다.

차원크기내용
Round4PREFLOP / FLOP / TURN / RIVER
Position2BB(player 0) / SB(player 1)
State (핸드강도)8PREMIUM→TRASH. 프리플랍은 Chen score, 포스트플랍은 treys 평가(7462 백분위)로 버킷팅
PrevAction4같은 라운드 내 상대의 직전 액션 압축: NONE / CHECK_CALL / SMALL_RAISE / BIG_RAISE
q[round][position][state][prev_action][action] = 4 × 2 × 8 × 4 × 8 = 2,048 셀
PrevAction 축이 핵심 — "상대가 큰 레이즈를 던졌는가"를 상태에 넣어 상대 행동에 조건부로 반응할 수 있게 한다.

2. 행동 (a) — 8종

FOLD, CHECK, CALL, RAISE_25, RAISE_50, RAISE_75, RAISE_100, RAISE_ALLIN (팟 대비 비율 베팅).

매 결정 시 legal_our_actions()로 합법 액션만 후보로 추린다.

3. 보상 (r) 과 리턴 (g) — 핵심 설계

최종 보상 r (terminal-only)

r = payoff = (최종 스택) − (시작 스택)

중간 스텝에는 보상이 없고, 핸드 종료 시 순손익(칩) 하나만 발생한다.

리턴 g — 비례 배분 (proportional credit assignment)

gi = ( investi / Σj investj ) × payoff
  • 크게 베팅한 결정일수록 그 핸드 결과에 대한 책임(credit)을 더 많이 받음
  • 투입이 0인 경우(전원 CHECK)는 payoff를 스텝 수로 균등 분배
  • CHECK_VIRTUAL_INVEST = 1 chip: CHECK도 투입 0이 아닌 1칩으로 간주 → CHECK 라인이 credit 0으로 흡수되어 학습이 고착되는 'CHECK=0 고정점' 문제 방지

4. 보상 전파 경로 — 몬테카를로 (MC)

에피소드 종료 후 일괄 업데이트 (부트스트래핑 없음):

Q(s, a) ← Q(s, a) + α · [ g − Q(s, a) ]
1 핸드 플레이 → trace에 (round, state, prevAction, action, invest) 누적 ↓ 핸드 종료 payoff 확정 → invest 비율로 g 배분 ↓ trace의 모든 스텝에 update_mc(g) 적용
왜 MC인가? TD의 maximization bias(낙관적 과대평가)가 포커의 분산 큰 보상에서 학습을 망가뜨린다. MC는 실제 핸드 결과만 평균내므로 그 편향이 없다. (GAMMA=0.9는 TD 경로 전용으로, MC 경로에선 미사용)

5. 학습 데이터 — 핵심 설계

고정된 알고리즘·하이퍼파라미터 위에서 매 에피소드 학습 상대 policy만 교체한다. 다른 변인을 모두 고정했기에 "상대 분포 다양화"의 효과만 분리해 측정하는 controlled comparison이 된다.

6. 평가 데이터 — 학습과 분리

평가는 탐색을 끈 raw greedy(best_action, argmax Q)로 수행.

  • 평가 상대 2종: ① 순수 Random ② 고정 TAG 룰베이스 (학습 상대와 분리해 일반화 측정)
  • 규모: 정밀 평가 100,000 games × 5회 반복(seed 1000~1004)으로 회차 SD 추정
  • 지표: mbb/g (milli-big-blinds per game) — 승률이 아닌 수익률. mbb = 평균 payoff × (1000 / BB)
  • 판정 기준(사전 등록): 평균 − 1·회차SD > 0 이 vs Random·vs TAG 양쪽 모두 성립해야 H1 채택
승률 대신 mbb/g를 쓰는 이유: 승률이 높아도 큰 팟을 잃으면 실제 수익은 적자일 수 있어(포커의 큰 분산 특성), mbb/g 양수 전환만을 학습 성공의 증거로 삼는다.

8. 탐색 함수 — Softmax (Boltzmann)

학습 중 행동 선택은 온도 기반 Softmax:

P(a) = exp( Q(a) / τ ) / Σa' exp( Q(a') / τ )
  • 초반 고온(τ=10) → 거의 균등 샘플링(탐색), 후반 저온(τ=0.5) → 거의 greedy(활용)
  • ε-greedy와 달리 Q값에 비례한 확률적 탐색 → 유망한 액션을 더 자주, 나쁜 액션도 가끔 시도
  • 수치 안정화를 위해 max_q를 빼고 지수화, 오버플로 시 greedy로 폴백
  • 탐색 함수로는 Softmax를 사용 (UCB 계열은 탐색이 학습을 잠식하는 경향이 있어 제외)
