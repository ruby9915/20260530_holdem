## 1. 강화학습 기초

**마르코프 결정 프로세스 (MDP)** — 순차 결정 문제의 수학적 틀. 구성요소는
상태(지금 어떤 상황인가), 행동(무엇을 할 수 있나), 전이(행동하면 상황이 어떻게
바뀌나), 보상(얼마나 잘했나), 할인율 γ(미래 보상을 얼마나 쳐줄까). "마르코프"의
뜻: 다음에 일어날 일이 *지금 상태*에만 의존하고 과거 경로에는 의존하지 않는다.
우리 연구에서: 홀덤 한 핸드, V2G의 하루, RTB의 예산 주기를 각각 MDP로 정식화했다.
"MDP 설계"란 이 다섯 요소를 도메인에 맞게 정의하는 일.

**tabular (표 기반)** — 상태×행동마다 칸이 하나씩 있는 표(Q-table)에 가치를
저장하는 방식. 신경망 근사와 달리 블랙박스가 없어 **모든 값을 직접 들여다보고
증명할 수 있다.** 우리가 tabular를 고집한 이유: Q(CHECK)=0.000이라는 지문을
정확히 관측하기 위해.

**Q-learning / Q값** — Q(s,a) = "상태 s에서 행동 a를 하고 그 뒤로 잘 플레이하면
기대되는 총보상". 학습은 경험으로 이 표를 채워가는 과정. greedy 정책 = 각 상태에서
Q가 가장 큰 행동을 고르는 것(argmax).

**Monte-Carlo(MC) 제어 vs TD** — MC는 에피소드가 *끝난 뒤* 실제 받은 총보상으로
Q를 갱신(우리의 PURE·PROP 둘 다 MC 계열). TD는 한 걸음마다 "지금 보상 + 다음
상태의 추정치"로 갱신. MC는 편향이 없지만 분산이 크고, TD는 그 반대 절충.

**편향(bias)과 분산(variance)** — 추정량이 참값에서 체계적으로 벗어나면 편향,
값이 들쭉날쭉하면 분산. **PURE(표준 MC) = 무편향·고분산, PROP(비례배분) =
저분산·편향 있음**. 우리 §5.3.1: PURE는 고분산 때문에 학습 후반에 붕괴한다.

**탐색-활용 (exploration-exploitation)** — 아는 최선을 쓸까(활용), 모르는 걸
시험할까(탐색)의 절충. **softmax 탐색**: Q값을 확률로 바꿔 좋은 행동일수록 자주,
나쁜 행동도 가끔 고른다. **온도(temperature)**: 높으면 고르게(탐색↑), 낮으면
argmax에 가깝게(활용↑). 우리 학습은 온도를 10→0.5로 서서히 낮춘다(감쇠). E8의
"탐색 강화" 처방 = 온도 하한을 2.0으로 올린 것 — ZCA에는 무효였다(방문할수록
0으로 *재고정*되므로).

**낙관적 초기화 (optimistic initialization)** — Q를 실제보다 높게(예: +50) 시작해
"모든 게 좋아 보이니 다 시험해보게" 만드는 탐색 기법. 방문할수록 참값으로 씻겨
내려가는 *일시적* 효과. 우리 Prop 1: ZCA의 0은 이것과 달리 *영구적*(방문해도 0으로
되돌아옴). E8 실증: 낙관 초기화는 ZCA를 못 고치고 학습 예산만 낭비시켰다(+184로
ID 훼손).

**고정점 (fixed point)** — 갱신을 반복해도 더 이상 움직이지 않는 값. 정상 학습의
고정점은 참값이다. ZCA의 핵심: 비례배분에서 Q(CHECK)의 고정점이 참값이 아니라
**정확히 0**이고, 이 0이 argmax 순위를 왜곡한다.

**seed (씨앗값)** — 난수 생성기의 시작값. 같은 seed면 같은 실험이 정확히 재현된다.
seed 하나의 결과는 우연일 수 있어(우리의 1칩 −376 사건) 여러 seed로 반복한다.

**함수근사 (function approximation)** — 표 대신 신경망 등으로 Q를 근사하는 것.
상태가 많을 때 필수지만 값이 셀 사이에 번져서(일반화) 우리의 "정확한 0" 지문이
"0 근방 구름"으로 흐려진다 — 4편 카드의 관문 ①이 이 문제.

**MAB (multi-armed bandit) / UCB** — 상태 없이 행동 선택만 반복하는 가장 단순한
학습 문제(슬롯머신 여러 대 중 고르기). UCB = "평균 보상 + 불확실성 보너스"로
낙관적으로 고르는 대표 알고리즘. 김선용 교수님 조명 논문(2022)이 이 방식.

## 2. 이 연구의 고유 개념

**기여도 배분 (credit assignment)** — 말단에 한 번 나온 성과를 "어느 행동 덕분
인가"로 나눠주는 문제. RL의 근본 난제 중 하나.

**비례배분 (PROP)** — 성과를 각 행동의 *투자액 비율*로 나누는 기여도 배분:
credit = (내 투자/총 투자) × 성과. 저분산이라 매력적이지만—

**ZCA (영-기여 흡수)** — 투자 0 행동(CHECK)은 비례배분에서 credit이 *항등적으로*
0 → Q가 참값과 무관하게 0에 고정 → 다른 행동들의 Q가 음수인 상태에서 greedy가
0인 CHECK로 빨려 들어감(**흡수**), 참값이 양수인 CHECK는 저평가됨(**은폐**).
이 양방향 왜곡이 **양방향 오순위**. 행동 결과가 **수동화**(턴 체크 65%).

**계열 정리 (Lemma 3)** — ZCA는 비례배분 특유가 아니라, 기여도 함수 φ가 비용 0
행동에 0을 주는 **모든** 비례 배분에서 발생(φ=투자액이든 공격성이든). 균등 배분과
표준 MC는 면역. "네가 만든 문제 아니냐" 반박의 방어선.

**VIC / 임계 가상비용** — CHECK에 가상의 투자 ε을 부여해 credit을 0에서 떼어내는
처방. **임계 ε_min = k·c/(1−k)**: 이보다 작으면 오순위 잔존, 충분히 크면 해소.
실증: 1칩=전이구간(2/6), 5칩+=유효(5/5). ※약어 주의: RL에 Variational Intrinsic
Control이라는 기존 "VIC"가 있어 논문에서는 약어 재고 중.

**지문 (fingerprint)** — 병리의 관측 가능한 표식. 우리 것: PROP+무개입이면
Q(CHECK)가 전 셀에서 **정확히 0.000** (자동 테스트로 고정). 적응 ε 카드에서는
이 지문이 제어 신호가 된다.

**전이구간** — 임계 근처에서 개입이 통할락 말락 하며 seed마다 부호가 요동하는
구간. 셀마다 팟이 달라 ε_min이 다르기 때문(1칩이 여기 있었다).

**용량-반응 (dose-response)** — 개입의 크기(용량)를 바꿔가며 효과(반응)를 보는
실험 설계. 우리 Table 2: −318 → −117 → +1230 → +1659의 단조 상승.

**ID / OOD** — 학습 분포 내(In-Distribution, 예: 학습 상대 TAG) / 분포 외(Out-Of-
Distribution, 예: 무작위 상대). **홀드아웃** = 학습에 안 쓰고 검증용으로만 남겨둔
상대/데이터.

**mbb/g** — milli big blind per game. 포커 성능 단위(payoff×500). +1000 = 게임당
빅블라인드 1개 이득.

**사전 등록 (pre-registration)** — 실험 전에 예측과 **기각 기준**을 문서로 박아
두는 것. 사후 합리화("어쩌다 맞은 걸 예측이라 우기기")를 차단. RTB 초안 P1~P6이
이 형식.

**credit 폴백 누수** — 우리가 발견·제거한 구현 인공물: 전부-체크 핸드에서 균등
분배로 폴백하던 코드가 비용 0 행동에 비(非)비례 credit을 흘려 허위 신호를 만듦.
"구현 인공물" = 알고리즘이 아니라 코드 세부가 만든 가짜 효과.

## 3. 보상 설계 이론

**보상 형성 (reward shaping)** — 학습을 돕기 위해 보상에 항을 추가하는 것.
**PBRS(potential-based reward shaping)**: 상태 퍼텐셜의 차이 꼴로만 추가하면
최적 정책이 안 바뀐다는 정리 — 그리고 이것이 **Q 초기화와 등가**라는 후속 정리
(우리 [6][7]). VIC가 이 틀의 어디에 있는지가 관련연구의 위치 설정.

**return-equivalence (RUDDER)** — 재분배된 보상의 총합이 원래 총보상과 같으면
(return-equivalent) 최적 정책 보존. 비례배분은 비용 0 행동에 대해 이 등가가
깨지고, ZCA는 정확히 그 비등가 영역에서 발생(보존 정리의 대우).

**action penalty** — 모든 행동에 일률적으로 작은 벌점을 주는 고전 기법(우리 [8]).
E8 실증: ZCA에는 무효 + 오히려 양 지표 훼손. VIC와의 차이 = **선택성**.

## 4. 협력게임·MARL (4편 카드 배경)

**MARL** — 다중 에이전트 강화학습. 여러 학습자가 한 환경에서 동시에 배운다.

**lazy agent (무임승차)** — 팀 보상을 전원이 똑같이 받으면 놀아도 보상이 오니
아무것도 안 하는 에이전트가 생기는 병. VDN(2017)이 명시한 동기.

**가치 분해 (VDN/QMIX)** — 팀 Q를 개인 Q들의 합(VDN)이나 단조 혼합(QMIX)으로
분해해 기여가 개인에게 귀속되게 하는 처방 계열.

**반사실 기여도 (counterfactual credit)** — "내가 다르게 했다면 팀 성과가 얼마나
달랐을까"로 기여를 재는 방식. difference rewards, COMA가 대표. **한계 기여
(marginal contribution)** = 내가 참여함으로써 늘어난 성과.

**Shapley value / null-player 공리** — 협력게임 이론의 공정 배분: 모든 연합에
대한 한계 기여의 가중 평균. null-player 공리 = "한계 기여가 0이면 배분도 0" —
그들에겐 공정성, 우리 렌즈로는 φ(기여0)=0, 즉 ZCA 계열의 씨앗.

**SQDDPG** — Shapley 몫을 각 에이전트의 critic으로 쓰는 MARL 알고리즘(AAAI
2020). 4편 카드의 본 무대.

**내재적 보상 (intrinsic reward)** — 환경 보상 외에 학습자가 스스로 만드는 보상
(호기심, 부지런함 등). Liu 2023의 diligence 보상 = 수동을 벌주는 방향(우리
처방과 반대) — "부지런함이 해로운 과제"가 승부처인 이유.

## 5. 정책 기울기·actor-critic (Zero Collapse 논의 배경)

**policy gradient (정책 기울기)** — Q표 없이 정책(행동 확률) 자체를 직접
미분해서 좋아지는 방향으로 조금씩 옮기는 방식. REINFORCE가 원형:
∇J = E[∇log π(a)·R]. 보상이 지역적으로 상수면 기울기가 0 — Zero Collapse의
"평탄 지역" 함정이 여기서 나온다(우리 ZCA와 기전이 다른 이유).

**actor-critic** — 정책(actor)과 가치 추정기(critic)를 함께 학습. critic이
actor의 학습 신호(advantage = Q − V)를 공급한다. **baseline** = 기울기의 분산을
줄이기 위해 보상에서 빼주는 기준값(보통 V). ZCA가 AC로 옮겨가는 경로: critic의
학습 target을 비례 credit으로 만들면 critic이 병들고 actor가 승계.

**DQN / DDPG** — 심층 Q-학습 / 연속 행동용 심층 actor-critic. 도메인 문헌에서
자주 등장하는 이름 정도로 알면 됨.

## 6. 최적화·제어 (적응 ε 카드 배경)

**폐형 추정 (closed-form)** — 반복 계산 없이 **공식 한 줄로 답이 나오는** 형태.
예: ε_min = k·c/(1−k)에 추정값을 꽂아 바로 ε*를 얻는 것. 반대말은 반복적·수치적
방법.

**라그랑주 승수 (Lagrange multiplier)** — "제약을 지키면서 최적화"를 푸는 고전
도구. 제약 위반에 매기는 **그림자 가격**으로 이해하면 된다: 제약을 어길수록
벌점이 세지고, 승수 자체를 조정해 "딱 제약을 만족하는 지점"을 찾는다. 적응 ε
카드에서: "지문(0-띠 점유율) ≤ 목표"를 제약으로 걸고 ε를 그 승수처럼 조정.

**SAC (Soft Actor-Critic)** — 보상 + 엔트로피(행동의 무작위성)를 함께 최대화하는
심층 RL의 대표 알고리즘. 우리가 참조하는 부분은 **자동 온도 조정**: 엔트로피
계수를 손으로 튜닝하지 않고, "목표 엔트로피"라는 제약을 라그랑주 승수로 걸어
자동으로 맞춘다 — 적응 ε의 선례 프레임.

**폐루프 적정 (closed-loop titration)** — (우리가 쓰는 표현) 측정 → 조정 → 재측정
루프로 용량을 자동으로 맞추는 것. 의학의 적정(약 용량을 반응 보며 조절)과 제어의
폐루프(출력을 되먹여 입력 조정)를 합친 그림. ε-적정 카드의 핵심 아이디어.

**메타그래디언트 (meta-gradient)** — 학습률·할인율·보너스 같은 *하이퍼파라미터
자체를* 바깥 루프에서 기울기로 학습하는 기법. 강력하지만 무겁고 분석이 어렵다 —
적응 ε의 후보 ③으로만 기록.

**oracle (완전정보 상한)** — 실제로는 모르는 정보(미래 부하, 상대 패)를 다 안다고
치고 푼 최적해. 도달 불가능한 상한선으로, 학습 정책이 "최적 대비 몇 %"인지 절대
좌표를 제공. 우리 평가 기준선 설계에 포함.

**LP / MIQP / MPC / PID** — 최적화·제어의 표준 도구들. LP=선형계획, MIQP=혼합정수
이차계획(현대차 V2B 논문이 이것), MPC=모델 예측 제어, PID=고전 되먹임 제어기.
공통점: **환경 모형을 알 때** 강력 — 그래서 "결정적 환경이면 RL 불필요" 비판의
근거이자, 우리가 불확실성을 명시해야 했던 이유.

## 7. 병리 진단 문헌의 개념들

**과대추정 편향 (overestimation bias)** — Q-learning의 max 연산이 잡음 속 최대값을
고르며 생기는 체계적 과대평가(Thrun & Schwartz 1993). 처방이 **Double Q-learning**
(추정과 선택을 두 표로 분리). 우리 장르 계보의 원형: 진단(1993) → 처방(2010).

**primacy bias** — 심층 RL이 초기 경험에 과적합해 이후 증거를 무시하는 병
(ICML 2022, 처방=주기적 reset). **dormant neuron** — 학습 중 뉴런이 점점 죽어
표현력이 줄어드는 병(ICML 2023, 처방=ReDo). 둘 다 "병리 명명→진단→최소 처방"
형식의 성공 사례 = 우리 논문의 형식적 선례.

**Zero Collapse** — 정책 기울기 입찰자가 0-보상 평탄 지역에서 기울기 소실로 갇히는
붕괴(2026). 우리와 표면은 같고 기전이 다름(정책 공간의 기하 vs 가치의 고정점).
그들의 완화책(초기화·스텝 제어·softplus·baseline)은 함정을 없애지 못하고 회피.

**reward hacking** — 에이전트가 보상의 허점을 파고들어 설계 의도와 다른 행동을
최적화하는 현상. 병리 정의·특성화 연구의 사례(NeurIPS 2022)로만 알아두면 됨.

## 8. 도메인 용어

**RTB (실시간 입찰)** — 광고 임프레션 단위의 실시간 경매. **1차/2차 가격 경매**:
낙찰자가 자기 입찰가를 내면 1차, 두 번째로 높은 입찰가를 내면 2차. **iPinYou** =
공개 입찰 로그 벤치마크. **ROAS/다중접점 귀속** = 광고 지출 대비 성과·전환 공을
접점별로 나누는 관행(지출-비례 귀속이 우리 PROP의 현실판). **예산 페이싱/
과소지출** = 예산을 기간에 고르게 쓰도록 조절 / 다 못 쓰는 문제(우리의 "입찰
중단 붕괴"가 이것의 credit-원인 버전).

**V2G / TOU / SMP / 열화비용** — 전기차↔전력망 양방향 충방전 / 시간대별 요금제
(경부하·중간·최대부하) / 계통한계가격(실시간 도매가) / 배터리를 쓸수록 닳는 비용
(방전의 invest로 쓴 항목).

**EH-IoT / Gilbert-Elliott / AoI / 대기전력** — 에너지 하베스팅(태양광 등 자가
충전) IoT / 좋음·나쁨 2상태를 오가는 고전 채널 모형 / Age of Information(정보가
얼마나 늙었나 — 침묵할수록 자동으로 벌점이 쌓여 우리 병리를 *회피*하는 정식화) /
sleep 중에도 새는 전력(3편에서 가상비용의 물리적 실체가 되는 값).

**GTO (근사 게임이론 최적)** — 포커에서 착취 불가능한 균형 전략. "외부·균형
상대 검증은 향후 과제"의 그 균형 상대.
