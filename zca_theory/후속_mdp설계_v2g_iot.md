# 후속 논문 MDP 설계서 — V2G(예비)·IoT(3편)

> **(2026-07-06 갱신)** 2편 = **RTB로 교체 확정** (기준: 지도교수 접점 제외, VIC 정합성만 —
> 지출-비례 귀속의 실재 관행 + 착취 구조 + ε_min 사전 계산). RTB의 MDP 설계·실험
> 사전 등록은 [`rtb_논문2_초안_v0.md`](rtb_논문2_초안_v0.md) §III~V. 아래 V2G 절은
> **예비 후보**로 강등하되 설계는 보존.

> 목적: 두 지도교수 합동 미팅 제출용. 1편(홀덤 ZCA+VIC)의 계열 정리가 예측하는 병리를
> 각 도메인에서 **정확히 어디에, 어떻게** 검증할지 보인다. 실험 전이므로 모든 성능 관련
> 서술은 **예측**이며, 예비 실험으로 확인 전까지 주장이 아니다.
>
> 공통 전제 3조건 (1편 계열 정리의 적용 조건):
> ① 성과가 말단에 한 번 나온다 (episodic terminal payoff)
> ② 행동이 성과와 같은 단위의 자원 투입이다
> ③ 투입 0 행동이 존재한다
>
> 공통 실험 규약 (1편 이식): ⑴ Q(무행동)≡0 지문 확인 → ⑵ 수동화의 행동 측정 →
> ⑶ ε 스윕(임계 미달/초과) → ⑷ 표준 MC(PURE) 동일 예산 3자 비교 → ⑸ 분포 이동
> 홀드아웃. 전 단계 학습 seed 6개, 정밀 평가 다회 반복.

## 2편 — V2G 스케줄링 (김선용 교수님 도메인)

**고유 연구 질문: 임계의 위치 예측.** 홀덤은 팟 규모 c가 셀마다 달라 ε_min 역산이
불가능했다(1편 한계 ①). V2G는 요금 구조가 균질하므로 **요금표에서 ε_min을 사전
계산하고, 실험이 그 위치에서 갈라지는지** 검증할 수 있다. 성공 시 이론이 "존재
예측"(1편)에서 "위치 예측"(2편)으로 승격된다.

### MDP

| 요소 | 설계 | 비고 |
|---|---|---|
| 에피소드 | 1일 = 24슬롯(1h), γ=1 | 청구 주기의 최소 단위 |
| 말단 보상 | 기준요금(무배터리) − 실제요금 [원] | 절감액. 슬롯 중간 보상 없음(전제 ①) |
| 상태 | 시간대 버킷 × SoC 버킷 × 요금구간 | 최소안 8×5×3=120셀(토이), 확장안 24×10×3×직전행동5=3,600셀(본실험) |
| 행동 (5) | 강충전 +2u / 약충전 +1u / **대기 0** / 약방전 −1u / 강방전 −2u | u = 배터리 용량의 1/10 |
| **불확실성 (RL 정당화)** | 건물 부하 = 시간대별 가우시안 요동, EV 가용성 = 도착·출발 확률 창(도착 시 잔량 무작위), (선택) SMP 실시간 단가 | **이게 없으면 MIQP·룰로 풀리는 결정적 문제** — RL을 쓰는 문헌(연구실 선행 포함)의 정당화 지점과 동일 |
| invest | 충전 = 구간단가×kWh, 방전 = 열화비용(원/kWh, V2G 문헌 표준값), **대기 = 0** | 전제 ②③ 충족 |
| 기여도 배분 | credit_t = invest_t / Σinvest × 절감액 | 1편 PROP과 동형 |

> **왜 RL인가 (예상 반박 선제).** 요금표가 알려진 결정적 설정이면 이 문제는
> 최적화(MIQP)·규칙으로 풀리며 학습이 불필요하다. 본 설계는 부하·EV 가용성을
> 확률화해 model-free 학습의 표준 정당화(분포를 모형 없이 경험에서 학습) 위에
> 선다. 논문의 주장도 "RL이 최선"이 아니라 **"V2G를 RL로 푸는 실재 파이프라인에서
> 말단 배분을 택하면 생기는 병과 그 처방"**이다. ε_min 사전 계산과의 정합:
> **문턱 = 아는 부분(요금 규모)의 함수, 정책 = 모르는 부분(부하·가용성 분포)의
> 함수** — 서로 충돌하지 않는다.

### ZCA 예측 (검증 대상 — 주장 아님)

- **지문**: PROP 학습 후 Q(대기)가 전 셀에서 정확히 0 (PURE는 분포).
- **흡수(수동화)**: 충·방전 Q가 잡음·음수인 셀에서 greedy가 대기로 붕괴 →
  **배터리 사이클 0회, 절감액 0** 에이전트. 홀덤 "턴 체크 65%"의 대응물.
  손해는 안 보므로 겉보기 정상 — 발견이 늦는 것까지 동형.
- **은폐**: 최대부하 직전 "대기하며 SoC 비축"의 참값 양수가 0에 고정 → 열등한
  조기 방전에 밀림.

### VIC와 ε_min 사전 계산

- invest(대기) ← ε. 해석 = 슬롯 용량의 기회비용.
- ε_min = k·c/(1−k)를 요금표로 사전 계산: c = 대표 슬롯 투자액(구간단가×u),
  μ 비율은 요금구간 차익에서 유도. **계산값을 먼저 논문에 박고 스윕이 그 좌우에서
  갈라지는지 본다** — 사후 맞춤(hindsight) 소거를 위해 사전 등록 방식.

### 평가

| 항목 | 설계 |
|---|---|
| 지표 | 절감액/일 [원], 배터리 사이클 수, Q(대기) 지문 |
| 기준선 | 무배터리 / 룰 정책(경부하 충전·최대부하 방전 — 1편 TAG 대응물) / 무작위 / PURE / **완전정보 최적화 상한(oracle: 실현된 부하·가용성을 알고 푼 LP·MIQP — 도달 불가 상한으로 학습 정책의 절대 위치 표시)** |
| 분포 이동 홀드아웃 | 미학습 요금제 — 계절 TOU 전환, 요금제 개편 시나리오 (1편 "무작위 상대" 대응물) |
| 데이터 | 한전 TOU 요금표·KPX SMP 시간별 단가(공개), 부하 프로파일은 가우시안 합성(연구실 선행 방식과 정렬) |

**연구실 접점**: 김선용 교수님 교신 2024 JKIECS(V2G tabular Q-learning)와 같은
도메인. 차이 = 그쪽은 슬롯별 보상 설계(병리 없음), 본 연구는 말단 배분 설계를
택할 때 생기는 병리와 그 처방. 선행 논문 인용으로 계보 접속.

## 3편 — 에너지 하베스팅 IoT 전송 스케줄링 (박승민 교수님 도메인)

**고유 연구 질문: 가상비용의 물리적 실현.** sleep의 대기전력(누설 전류)은 실재
비용이므로, VIC = "비용 0이라는 추상화가 버린 실제 비용의 복원"이 된다. 가상비용이
가상이 아니게 되는 첫 도메인 + 완전 통제 합성 환경에서 용량-반응 곡선 정밀 측정.

### MDP

| 요소 | 설계 | 비고 |
|---|---|---|
| 에피소드 | 1일 = 48슬롯(30min), γ=1 | 에너지 중립 운용의 보고 주기 |
| 말단 보상 | Σ전달 효용 − 블랙아웃 페널티 | 주기 말 일괄 평가(전제 ①) |
| 상태 | 하베스트 국면 버킷 × 배터리 버킷 × 채널 상태 × 큐 길이 | 6×5×2×3 = 180셀 (Gilbert-Elliott 2상태 채널) |
| 행동 (3) | 고출력 송신(성공률 高·에너지 大) / 저출력 송신(低·小) / **sleep 0** | |
| invest | 송신 에너지 [mJ], **sleep = 0** | 전제 ②③ 충족 |
| 기여도 배분 | credit_t = invest_t / Σinvest × 말단 효용 | 1편 PROP과 동형 |

> **왜 RL인가.** 채널 전이확률과 하베스트 분포는 노드 입장에서 미지이며 배치
> 환경마다 달라, 모형 기반 최적화보다 경험 학습이 자연스러운 설정이다(EH-IoT
> RL 문헌의 표준 정당화). V2G와 마찬가지로 평가에는 실현 경로를 알고 푼
> 완전정보 상한(oracle)을 함께 둔다.

### ZCA 예측 (검증 대상 — 주장 아님)

- **지문**: Q(sleep) 전 셀 정확히 0.
- **흡수(수동화)**: 채널 악화 구간에서 송신 Q 전부 음수 → 영구 sleep →
  **데이터 블랙아웃(보고 중단 노드)**. 가장 유형적인 실패 양상.
- **은폐**: 하베스트 피크 직전 "sleep으로 배터리 비축"의 참값 양수가 0에 고정 →
  낭비적 송신.

### VIC

- invest(sleep) ← ε = **대기전력 × 슬롯 시간** (데이터시트 실측치 사용).
- 3편 고유 기여: ε의 값이 자유 매개변수가 아니라 물리 상수로 고정됨 →
  "임계를 넘는가"가 하드웨어 사양의 함수가 되는지 검증.

### 평가

| 항목 | 설계 |
|---|---|
| 지표 | 전달률, 블랙아웃률, 에너지 효율 [bit/mJ], Q(sleep) 지문 |
| 기준선 | 임계값 룰 정책(EH-IoT 문헌 표준) / greedy / 무작위 / PURE |
| 분포 이동 홀드아웃 | 미학습 채널 전이확률(G↔B), 미학습 하베스트 국면(맑음 학습→흐림 평가) |
| 데이터 | 전부 합성(Gilbert-Elliott 채널 + 태양광 트레이스; 필요 시 NREL 공개 일사량) — 전 매개변수 통제로 단일변수 방법론 보존 최고 |

## 두 설계의 역할 분담

| | V2G (2편) | IoT (3편) |
|---|---|---|
| 이론 승격 | 임계 **위치** 예측 검증 | ε의 **물리적 실체** 검증 |
| 환경 성격 | 공개 실데이터(요금표) 기반 | 완전 통제 합성 |
| 실패 양상 | 배터리 미사용(절감 0) | 데이터 블랙아웃 |
| 예비 실험 우선순위 | 토이(120셀)에서 지문·수동화 재현 확인 | 180셀에서 지문·수동화 재현 확인 |

## 대안 도메인 검토 — "왜 하필 이 둘인가" (2026-07-06 조사)

RL 정당성은 구조로 세 등급이다: **① 상대가 있는 환경**(상대 전략 미지 — 홀덤·입찰)
> **② 미지의 확률 환경**(분포를 경험에서 학습 — 본 설계 둘 다 여기) > **③ 알려진
환경**(최적화로 풀림 — RL 부적합). 후보 검증 결과:

| 후보 | RL 정당성 | ZCA 적합 | 비고 |
|---|---|---|---|
| EH-IoT (3편) | ②, 문헌 표준 (Blasco et al., IEEE TWC 2013: 미지 Markov 과정에 사전정보 없는 Q-learning) | 좋음 | 채택 |
| V2G (2편) | ② (불확실성 명시 시; EV 충전 model-free DRL 문헌 표준과 일치) | 좋음 + ε_min 사전계산 | 채택 |
| **실시간 입찰(RTB)** | **① — 홀덤과 동형**(경쟁 입찰자=상대). Cai et al. WSDM 2017, iPinYou 공개 실데이터 | 매우 좋음 (no-bid=투입 0, 예산주기 말 성과, 지출-비례 ROAS 관행 실재) | **예비/4편 후보** — 지도교수 접점 없음이 유일한 약점 |
| 인지무선 스펙트럼 접근 | ② | 조건 ①(말단 보상) 재설계 필요(통상 per-step) | 보류 |

## 신규성 조사 — RTB·IoT 유사 선행 (2026-07-06, 다각도 웹 검증)

**결론: 두 도메인 모두 직접 충돌(비용-비례 MC credit + 무행동 0-고정점 + 가상비용 처방의 결합) 없음.**
단, 반드시 인용·차별화할 근접 논문과 두 가지 횡단 위협이 확인됨.

**공통 인용·차별화 대상:**
- Kumar et al., "Zero Collapse" (arXiv:2605.30896, 2026) — 경매 입찰 RL에서 0-보상 평탄 지역에
  갇히는 붕괴. **가장 근접**하나 기전이 다름: policy gradient 소실(정책 공간) vs 본 연구의
  0-credit 고정점(가치 공간). 처방도 초기화/구조 vs 가상비용.
- Tanimoto et al. (Front. Behav. Neurosci. 2020) — 쥐의 무행동에 "절약 비용=은닉 보상 ψ" 부여
  Q-learning 모형. VIC의 거울상(부호 반대, 신경과학). 교차 도메인 인용 가치.
- Liu et al., "Lazy Agents" (ICML 2023) — 무행동 병리의 MARL 친척(기전 다름: 팀 보상 공유).

**RTB 특이:** Cai et al. RLB(WSDM 2017)는 bid=0이 행동 공간에 있으나 정확한 DP라 추정 병리
없음(인용: "no-bid는 정당한 행동" 선례). Wu et al. DRLB(CIKM 2018)의 "reward design trap"은
입찰 RL 보상 설계 병리의 선례(기전 다름). 지출-비례 MC credit을 쓴 RTB 논문은 미발견.

**IoT 특이:** ACES(Fraternali et al., TOSN 2020)가 1순위 주의 — sleep일수록 보상 0인 행동 지수
보상 + 누설전류를 *환경 모델*에 반영(보상/credit에는 없음, 병리 진단 없음). 반드시 인용·차별화.
AoI 계열은 위치설정 위협: 침묵에 비용이 자동 누적돼 병리가 안 생기는 정식화 — "AoI로 풀면
되잖아" 심사 질문에 "per-slot 정식화는 병리를 회피, 말단 배분 정식화에선 구조적으로 발생"으로
답할 것. 말단 보상+MC 역전파 EH-IoT 논문 미발견(전부 per-slot).

**횡단 위협 2건:**
1. **자기설정 반론** — 두 도메인 모두 그 credit 방식을 아무도 안 쓴다는 사실이 신규성인 동시에
   "스스로 만든 문제" 반론 근거. 방어: 1편 계열 정리 + RTB의 지출-비례 귀속(ROAS) 실재 관행 +
   말단 평가의 자연성(전환 지연·일일 보고). RTB가 이 방어에 가장 강함.
2. **★약어 충돌** — "VIC"는 RL에서 Variational Intrinsic Control(Gregor et al. 2016)로 확립된
   약어와 충돌. "ZCA"는 ZCA whitening(비RL)과 충돌(덜 위험). 1편 투고 전 명칭 재고 필요.

## 위험 요인 (정직)
