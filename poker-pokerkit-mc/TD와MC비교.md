# TD(0) vs MC + 즉각 비용 — 설계 및 비교 분석

## 1. 실험 환경

두 모델 모두 동일한 환경에서 실행된다. PokerKit 라이브러리로 구현된 헤즈업(1대1) 노리밋 텍사스 홀덤 단판 구조이며, 양쪽 플레이어 모두 매 판 시작 시 200 칩으로 초기화된다. 블라인드는 스몰 1, 빅 2로 고정한다.

평가는 200 에피소드마다 수행되며, 무작위로 행동하는 RandomAgent와 라운드·핸드 강도·베팅 직면 여부에 따라 고정된 정책을 쓰는 RuleBasedAgent를 상대로 각각 200 게임씩 대전하여 승률과 평균 payoff(mbb/g, 표준오차 포함)를 기록한다. 총 학습 에피소드는 40,000이며 학습률 α=0.1, 할인율 γ=0.9를 사용한다. 탐색 비율 ε는 처음에 1.0(완전 랜덤)에서 시작해 학습이 80% 진행되는 시점(32,000 에피소드)까지 선형으로 0.05까지 감소한다.

### 포지션 차원 추가 및 학습 중 교대

학습과 평가 모두 매 에피소드마다 학습 에이전트의 포지션을 교대한다. 짝수 에피소드에서는 BB(빅블라인드), 홀수 에피소드에서는 SB(스몰블라인드)로 게임을 시작한다. 동시에 포지션을 Q-테이블의 상태 차원으로 추가해 SB와 BB의 경험이 서로 다른 셀에 저장되도록 한다.

포지션을 구분하는 이유는 두 가지다. 첫째, 헤즈업에서 SB와 BB는 행동 순서, 강제 베팅 금액, 정보 우위가 모두 다르므로 사실상 다른 게임이다. 같은 패라도 포지션에 따라 최적 행동이 달라진다. 둘째, 평가도 한쪽 포지션에만 고정하면 상대(예: RuleBased)가 다른 포지션에서 약한 패로 자주 폴드해 학습 에이전트가 행동하지 않고도 블라인드 차익을 챙기는 인공적인 결과가 나온다.

실제로 구현 초기에 포지션을 BB 한쪽으로 고정했을 때 학습되지 않은 에이전트(모든 Q값 0, 따라서 항상 FOLD)가 RuleBased 상대로 83%의 승률을 보였다. 이는 SB 차례인 RuleBased가 약한 패로 매번 먼저 폴드해 BB가 SB 블라인드 1칩을 자동으로 가져가는 효과 때문이었다. 포지션 교대 적용 후 같은 학습되지 않은 에이전트의 vs RuleBased 승률은 약 42%로 정상화되었고, 평균 payoff도 음수로 측정된다.

### RuleBasedAgent 정책 구조

평가 기준선이 되는 RuleBasedAgent는 두 가지 축으로 행동을 결정한다.

**첫째, 라운드별로 다른 행동을 한다.** 단순히 "STRONG이면 팟 50% 베팅"으로 통일하면 프리플랍처럼 팟이 작을 때(예: SB-BB가 1-2면 팟 4) 50% 베팅은 겨우 2칩에 불과해 의미있는 압박이 되지 않는다. 따라서 라운드별로 베팅 비율을 다르게 적용해 절대 금액의 일관성을 맞춘다. 프리플랍에서는 팟이 작으므로 큰 비율로(STRONG이면 100%), 리버에서는 팟이 충분히 크므로 작은 비율로(STRONG이면 50%) 베팅한다.

**둘째, 베팅을 직면했는지에 따라 다른 정책을 쓴다.** 체크가 가능한 상황(아무도 베팅하지 않음)에서는 강한 패로 능동적으로 베팅을 열고, 약한 패는 무료로 다음 카드를 본다. 콜할 금액이 있는 상황에서는 강한 패로 3-bet 레이즈, 좋은 패는 콜, 약한 패는 폴드로 응답한다.

| 라운드 | PREMIUM (오픈/직면) | STRONG | GOOD | DECENT | MEDIOCRE | WEAK | POOR | TRASH |
|---|---|---|---|---|---|---|---|---|
| PREFLOP | RAISE_100 / RAISE_100 | RAISE_75 / RAISE_75 | RAISE_50 / CALL | RAISE_25 / CALL | CHECK / CALL | CHECK / FOLD | CHECK / FOLD | CHECK / FOLD |
| FLOP    | RAISE_100 / RAISE_100 | RAISE_75 / RAISE_75 | RAISE_50 / CALL | RAISE_25 / CALL | CHECK / FOLD | CHECK / FOLD | CHECK / FOLD | CHECK / FOLD |
| TURN    | RAISE_75 / RAISE_75   | RAISE_75 / RAISE_50 | RAISE_25 / CALL | CHECK / CALL    | CHECK / FOLD | CHECK / FOLD | CHECK / FOLD | CHECK / FOLD |
| RIVER   | RAISE_75 / RAISE_75   | RAISE_50 / RAISE_50 | RAISE_25 / CALL | CHECK / CALL    | CHECK / FOLD | CHECK / FOLD | CHECK / FOLD | CHECK / FOLD |

이 정책은 포커 이론을 완전히 반영하지는 않지만, 학습된 에이전트가 단순히 항상 폴드하거나 항상 올인하는 수준 이상의 전략을 학습해야 이길 수 있는 적당한 난이도의 기준선을 제공한다.

## 2. 상태 추상화 구조

Q-테이블의 인덱스는 세 가지 차원의 조합으로 구성된다. 차원을 하나 추가하면 해당 수가 곱해지므로, 테이블 크기는 덧셈이 아니라 곱셈으로 늘어난다.

### 베팅 라운드 (4가지)

프리플랍, 플랍, 턴, 리버로 구분한다. PokerKit의 street_index 값을 그대로 매핑한다.

### 핸드 강도 버킷 (8가지)

에이전트가 보유한 패의 강도를 8단계로 추상화한다. 상위 12.5%부터 하위 12.5%까지 균등한 폭으로 8구간으로 나눈다.

- PREMIUM:  최상위 12.5% 패 (AA, KK, AKs 등)
- STRONG:   상위 12.5~25%
- GOOD:     상위 25~37.5%
- DECENT:   상위 37.5~50%
- MEDIOCRE: 상위 50~62.5%
- WEAK:     상위 62.5~75%
- POOR:     상위 75~87.5%
- TRASH:    하위 12.5% 패 (72o 등)

프리플랍에서는 두 장의 홀카드를 Chen formula로 점수화하여 8구간으로 나눈다. Chen formula는 포커 핸드의 사전 강도를 수치화하는 고전적인 방법으로, 높은 카드 값, 페어 여부, 슈티드 여부, 갭(두 카드 간 숫자 차이)을 종합해 점수를 계산한다. 예를 들어 AA는 최고 점수, 72o는 최저 점수에 해당한다.

플랍 이후에는 treys 라이브러리의 Evaluator를 사용해 현재 핸드가 가능한 모든 핸드 중 상위 몇 퍼센트에 해당하는지 계산하고, 이를 8구간(12.5% 단위)으로 나눈다.

### 행동 (8가지)

폴드, 체크, 콜, 팿의 25%/50%/75%/100% 레이즈, 올인으로 구성된다. 핸드 강도와 동일하게 8가지로 맞춰 (State, Action) 조합 테이블을 정사각형으로 유지한다.

### 포지션 (2가지)

SB(스몰블라인드)와 BB(빅블라인드)를 구분한다. 헤즈업에서 두 포지션은 행동 순서와 강제 베팅 금액이 달라 사실상 다른 게임이므로, Q-테이블이 두 경우를 별도 셀에 저장해야 포지션별 최적 정책을 학습할 수 있다.

### 현재 테이블 크기

라운드 4개 × 포지션 2개 × 핸드강도 8개 × 행동 8개 = **512 엔트리**

핸드 강도와 행동의 차원을 같은 수(8)로 맞춰 (State, Action) 테이블이 정사각형이 되고, 두 축의 해상도가 동일해 테이블을 시각화했을 때 가독성이 좋다.

## 3. 상태 확장 논의

### 현재 모델이 모르는 정보

현재 에이전트는 자신의 패 강도, 라운드, 포지션만 알고 있다. 스택 크기, 상대가 이번 라운드에 어떤 행동을 했는지, 콜할 금액이 팟의 몇 %인지 등은 모른 채 행동을 결정한다.

### 포지션 차원 (구현 완료)

헤즈업에서 포지션은 단순한 순서 구분이 아니다. 프리플랍에서는 스몰블라인드(SB)가 먼저 행동하므로 상대의 의도를 모르는 채 결정해야 하고, 빅블라인드(BB)는 상대의 콜/레이즈를 본 뒤 결정한다. 플랍 이후에는 반대로 BB가 먼저 행동하므로 SB가 정보 우위를 갖는다. 즉, SB인지 BB인지에 따라 최적 전략 자체가 달라진다.

본 구현은 포지션을 이미 상태 차원에 포함시켜 Q-테이블이 두 포지션을 별도 셀에 저장한다. 결과적으로 테이블 크기는 기본 256 엔트리에서 **512 엔트리**로 확장되었고, 에이전트는 "내가 SB일 때"와 "BB일 때"의 정책을 독립적으로 학습한다.

### 상대 콜 압박 차원 (향후 과제)

현재 내가 얼마를 내야 하는지를 팟 대비 비율로 12구간으로 나누어 상태에 포함하는 방식이다. 체크 상황(0원), 소액 콜, 중간 레이즈, 대형 레이즈 등으로 구분해 에이전트가 압박 수준에 따라 다른 전략을 학습하게 할 수 있다. 상대가 폴드한 경우는 게임이 이미 종료되므로 이 차원에 포함할 필요가 없다.

콜 압박을 추가했을 때 테이블 크기: 4 × 2 × 8 × 12 × 8 = **6,144 엔트리**

#### 단계적 연구 계획

본 실험은 두 단계로 분리해서 진행한다.

**1단계 (본 발표)**: 512 엔트리(라운드 × 포지션 × 강도 × 행동) 환경에서 TD(0)와 MC + 즉각 비용의 학습 곡선을 비교한다. 두 알고리즘의 reward 전파 구조 차이가 작은 상태 공간에서 어떻게 드러나는지 확인하는 것이 목표다.

**2단계 (후속 연구)**: 콜 압박 차원을 추가해 6,144 엔트리로 확장한 뒤, 동일 알고리즘 비교를 반복한다. 상태 공간이 12배 커졌을 때 두 알고리즘의 격차가 어떻게 벌어지는지가 핵심 관찰 대상이다.

#### 콜 압박 추가 시 필요 학습량의 이론적 추정

학습이 안정화되려면 각 (상태, 행동) 셀이 평균 수백 회 방문되어 Q값 분산이 충분히 줄어들어야 한다. 현재 512 엔트리에 40,000 에피소드를 사용하는 설정은 평균 방문 횟수로 환산하면

$$\bar{n}_{512} \approx \frac{40{,}000 \times 7}{512} \approx 547 \text{ 회/셀}$$

이다 (에피소드당 평균 액션 수 7로 가정). 6,144 엔트리에서 동일한 평균 방문 횟수를 유지하려면

$$N_{6144} \approx \frac{547 \times 6{,}144}{7} \approx 480{,}000 \text{ 에피소드}$$

가 필요하다. 이는 **MC + 즉각 비용**의 경우에 해당하는 추정이다. 한 에피소드에 전체 경로가 한 번에 역방향 전파되므로 셀 커버리지가 곧 학습량으로 직결된다.

**TD(0)는 같은 커버리지로는 부족하다.** reward 신호가 한 스텝씩만 전파되므로 게임 초반 액션까지 신호가 닿으려면 동일한 (state, action) 셀을 추가로 수십 회 더 방문해 다음 상태의 Q값이 안정된 뒤에야 의미있는 업데이트가 이루어진다. 평균 7스텝 게임에서 프리플랍 액션까지 전파하려면 셀당 방문 횟수가 수천 회 단위로 필요하므로, 6,144 엔트리에서 TD가 MC와 비슷한 수준으로 수렴하려면 **500만~1,000만 에피소드** 규모가 필요할 것으로 추정된다.

| 알고리즘 | 6,144 엔트리에서 추정 필요 에피소드 |
|---|---|
| MC + 즉각 비용 | 약 48만 (커버리지 기준) |
| TD(0) | 약 500만 이상 (전파 안정화 포함) |

이 격차 자체가 본 프로젝트가 제안하는 MC + 즉각 비용 설계의 정량적 근거가 된다. 즉 1단계에서 두 알고리즘 차이를 확인하고, 2단계에서는 "MC만 현실적인 시간 내에 수렴, TD는 사실상 불가" 라는 강한 비교 결과를 얻는 것이 후속 연구의 목표다.

#### 2단계에서 추가로 보고할 지표

1단계와 동일한 mbb/g, 표준오차, vs RuleBased 승률에 더해 다음을 추가 기록한다.

- **셀 커버리지**: 학습 종료 시 전체 6,144 엔트리 중 한 번이라도 방문된 셀의 비율
- **셀당 평균 방문 횟수 분포**: 히스토그램으로 시각화해 두 알고리즘이 어느 영역에서 학습 부족인지 비교
- **wallclock 시간**: TD가 같은 mbb/g 수준에 도달하는 데 걸린 실제 시간 vs MC

### 테이블 크기와 두 알고리즘의 관계

테이블이 커질수록 각 셀이 방문되는 빈도가 줄어든다. TD(0)는 같은 (상태, 행동) 조합을 여러 번 방문해야 reward 신호가 앞쪽 액션까지 전파되므로 방문 빈도 감소에 취약하다. MC는 한 에피소드에서 전체 경로를 한 번에 역방향으로 전파하므로 테이블이 커져도 상대적으로 덜 영향을 받는다. 본 실험에서는 포지션 차원과 핸드 8단계를 모두 적용해 테이블이 원래 크기의 3.2배(160 → 512)가 되었으므로, 두 알고리즘 간 수렴 속도 차이가 더 뚜렷이 드러날 것으로 예상된다.

| 상태 공간 크기 | TD(0) 수렴 | MC 수렴 |
|---|---|---|
| 160 (포지션 미포함, 강도 5단계) | 가능하지만 느림 | 빠름 |
| 320 (+포지션) | 더 느려짐 | 약간 느려짐 |
| **512 (현재 · +강도 8단계)** | **매우 느림** | **수렴 가능** |
| 6,144 (+콜압박, 향후) | 40,000 에피소드 내 수렴 어려움 | 수렴 가능 |

## 4. 두 알고리즘의 동작 방식

### TD(0) 방식

매 액션 직후에 Q값을 업데이트한다. 게임 진행 중에는 reward를 0으로 놓고 다음 상태의 Q값을 이용해 현재 Q값을 보정하는 bootstrap 방식을 쓴다. 게임이 끝날 때 마지막 액션에만 실제 payoff(최종 칩 증감)를 전달한다.

예를 들어 프리플랍에서 레이즈 → 플랍에서 콜 → 턴에서 레이즈 → 쇼다운 이라는 경로로 이겼다면, 승리 신호는 마지막 턴 레이즈에만 즉시 전달된다. 프리플랍 레이즈가 이 승리에 기여했다는 사실은 이후 반복 방문을 통해 간접적으로만 학습된다.

또한 베팅할 때 칩이 즉각 줄어드는 비용이 reward에 직접 기록되지 않는다. 이론적으로는 패배한 에피소드의 음수 payoff가 마지막 액션에서 시작해 한 스텝씩 앞으로 새어 나오면서 베팅 비용이 간접적으로 학습되지만, 같은 (상태, 액션) 셀을 수천 번 방문해야 안정화된다. 학습 초중반에는 이 전파가 충분히 일어나지 않아 큰 베팅도 Q값이 양수 방향으로 튀는 경향이 있다.

### MC + 즉각 비용 방식

매 액션마다 칩 변화량(베팅 시 음수, 체크/폴드 시 0)을 기록해두고, 게임이 끝난 후 역방향으로 누적 리턴 G를 계산하여 전체 액션을 한꺼번에 업데이트한다.

같은 예시에서, 게임이 끝난 후 턴 레이즈 → 플랍 콜 → 프리플랍 레이즈 순으로 역방향 전파된다. 즉 첫 번째 액션인 프리플랍 레이즈도 이번 에피소드 단 한 번으로 승리 기여분을 학습받는다.

베팅 비용도 즉시 반영되므로 "이 상황에서 팟을 키우는 것이 이익인가 손해인가"를 더 정확하게 판단하게 된다. FOLD 역시 칩 변화가 0이지만, 이전 액션들의 G 계산에 자연스럽게 포함된다.

### 업데이트 수식 비교

두 알고리즘의 본질적 차이는 Q값 업데이트 식 한 줄에 압축된다.

**TD(0)** — bootstrap 사용:

$$Q(s,a) \leftarrow Q(s,a) + \alpha \big[\, r + \gamma \underbrace{\max_{a'} Q(s', a')}_{\text{← bootstrap}} - Q(s,a) \,\big]$$

오른쪽의 $\max_{a'} Q(s', a')$는 **실제 미래 보상이 아니라 현재 Q-테이블에서 읽어온 추정값**이다. 학습 초기에는 이 값도 부정확하지만, 그 부정확한 값으로 또 다른 Q를 갱신한다. 반복을 통해 점점 자기 보정된다.

**MC** — bootstrap 미사용:

$$Q(s,a) \leftarrow Q(s,a) + \alpha \big[\, G_t - Q(s,a) \,\big]$$

여기서 $G_t = r_t + \gamma\, r_{t+1} + \gamma^2 r_{t+2} + \cdots$ 는 **에피소드가 끝날 때까지 실제로 받은 보상의 누적합**이다. 추정치를 일절 끼지 않고 진짜 결과만 사용한다.

> **요약**: TD는 "추정으로 추정을 업데이트"(bootstrap), MC는 "실제 결과로만 업데이트". 이 한 가지 차이가 표 5절의 모든 항목(전파 속도, 분산, 편향)을 결정한다.

## 5. 핵심 차이 요약

| 항목 | TD(0) | MC + 즉각 비용 |
|---|---|---|
| 업데이트 시점 | 매 액션 직후 | 에피소드 종료 후 일괄 |
| 베팅 비용 반영 | 즉시 반영 없음 (반복 방문으로만 간접 전파) | 액션 즉시 반영 |
| 승리 신호 전파 | 반복 방문이 필요 | 에피소드 1회에 전체 전파 |
| FOLD 학습 신호 | 없음 | 이전 액션의 G에 포함 |
| Bootstrap 사용 | 사용 (다음 상태 Q값 참조) | 미사용 (실제 결과만 사용) |
| 분산 | 낮음 | 높음 (전체 경로 의존) |
| 편향 | 높음 (근사값 누적) | 낮음 (실제 수익 그대로) |

## 6. 예상 비교 결과

학습 초반에는 MC가 더 빠르게 승률이 올라갈 것으로 예상된다. TD(0)는 같은 경로를 여러 번 밟아야 학습이 전파되므로 초반에 뒤처진다. 에피소드가 충분히 쌓이면 두 방식 모두 수렴하여 최종 성능은 유사해질 수 있다.

상태 공간을 확장할수록 이 차이가 더 명확하게 나타난다. TD(0)는 방문 빈도가 낮아 수렴이 어려워지는 반면, MC는 에피소드 1회에 전파하는 구조 덕분에 상대적으로 영향이 작다.

FOLD의 Q값에서도 차이가 드러난다. TD(0)에서는 FOLD에 대한 reward 신호가 없으므로 Q값이 0 근처에 머무르는 경향이 있다. MC에서는 FOLD를 선택한 에피소드의 실제 결과가 반영되므로 상황에 따라 유의미한 양수 또는 음수 값을 갖는다. 즉, 약한 패일 때 폴드하는 것이 얼마나 이익인지를 수치로 학습할 수 있다.

## 8. 발표 서술 방향

이 프로젝트는 다음의 서사 구조를 따른다.

먼저 TD(0)를 포커에 직접 적용했을 때 두 가지 구조적 문제가 발생함을 확인했다. 첫째, 베팅이라는 즉각적인 비용이 Q값에 전혀 반영되지 않는다. 둘째, 게임 초반 액션의 기여도가 학습되려면 동일한 경로를 수백 번 반복해야 한다.

이를 해결하기 위해 Monte Carlo 방식과 즉각 비용 전파를 결합한 새 알고리즘을 설계했다. 에피소드 한 번으로 전체 경로를 역방향 전파하고, 베팅 순간의 칩 감소를 reward로 기록함으로써 두 문제를 동시에 해결한다.

마지막으로 환경, 추상화, 하이퍼파라미터를 모두 동일하게 통제한 상태에서 알고리즘만 바꿔 학습 곡선을 비교하여, 설계 변경이 실제로 효과가 있는지 실험으로 검증한다.
